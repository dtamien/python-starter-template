# Contributing

Thanks for your interest in improving this project. Contributions are welcome.

## How to contribute

- Open an issue to discuss significant changes or questions.
- Fork the repo, create a feature branch from `main`, and submit a pull request.
- Follow branch naming conventions if applicable (e.g., `feature/<name>`).

## Code style

Follow the existing project style (see [Taskfile.yml](./Taskfile.yml)).

## Tests

Add or update tests for new behavior and ensure all tests pass locally before opening a PR.

## Commit messages

Use clear, descriptive commit messages.

## Review process

Be responsive to review feedback. Maintainers may request changes.
